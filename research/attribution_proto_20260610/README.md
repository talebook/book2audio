# 说话人识别 L1 规则层原型验证（2026-06-10）

测试样本：`book/chat.txt`（《玄鉴仙族》第三章节选，14 条对白，含多角色对话），金标为人工标注。

## 结果

```bash
uv run python research/attribution_proto_20260610/proto_rules.py book/chat.txt
```

| 指标 | 数值 |
|---|---|
| 规则覆盖率（给出归属） | 13/14 = 93% |
| **对照金标准确率** | **10/14 = 71%** |

规则命中分布：R3 邻段主语 8 条（主力）、R1/R2 同段名字+动词 2 条、同段沿用 1 条、R4 轮替 3 条（其中 2 条错）。

## 失败模式（正好对应需要 L2 模型的场景）

| 失败 case | 原因 | 解法 |
|---|---|---|
| "害，项平哥。"→田芸 | 说话人名字在数段之后才首次出现 | CSI 模型（带长上下文）|
| "X真是好福气！"→田芸的父亲 | 称谓/亲属表达，非直接人名 | CSI 模型 / 称谓归一化 |
| "好男丁…"→李木田 | 代词"他"需回指消解 | CSI 模型 |
| "来喽！"→李通崖 | 新角色全文仅出场1次，没进人名清单 | NER 替代词频法建人名清单 |

## 结论

1. 规则层在难样本上可达 ~70% 准确率，简单样本（对白带"X说"较多的网文）预计更高，作为 L1 成本为零，值得保留
2. **人名清单构建是规则层的地基**——词频法太脆，应换成 NER（如 hanlp/LTP 或 CSI 自带的 mention 识别）
3. 错误集中在 代词回指/远距离归属/称谓表达，与选型报告（[[tech_reselect_20260610]]）的判断一致：**必须上 L2 CSI 模型兜底**，规则只在高置信线索（R1/R2/R3）时直接采纳
4. R4 轮替规则风险高（2/3 错），建议降级为"仅在已确认的双人对话场景中使用"
